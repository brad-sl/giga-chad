# Dashboard Backend - AI Orchestration Platform

A Flask-based REST API backend for monitoring and managing AI orchestration runs, proposals, and executions.

## Quick Start

### Prerequisites

- Python 3.8+
- pip or pipenv

### Installation

1. Navigate to the dashboard directory:
```bash
cd projects/dashboard
```

2. Create a virtual environment:
```bash
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

3. Install dependencies:
```bash
pip install -r requirements.txt
```

### Running the Server

```bash
python app.py
```

The server will start on `http://localhost:5000` and automatically initialize the SQLite database.

## Database

- **Location:** `orchestrator.db` (created in the project root)
- **Type:** SQLite 3
- **Auto-initialization:** The database schema is created on first run

### Database Initialization Commands

```bash
# Initialize database from Python
python -c "from app import app; from database import init_db; init_db(app)"

# Or use Flask CLI
export FLASK_APP=app.py
flask init

# Seed sample data
flask seed

# Reset database (⚠️ deletes all data)
flask reset
```

## API Endpoints

### Runs

- **GET `/api/runs`** - List all runs
  - Query params: `limit`, `offset`, `status`
  - Returns: List of run objects with pagination

- **POST `/api/runs`** - Create a new run
  - Body: `{ agent_name, agent_type, status?, result? }`
  - Returns: Created run object

- **GET `/api/runs/{run_id}`** - Get run details
  - Returns: Run object with related proposals, executions, audit logs

### Proposals

- **GET `/api/proposals`** - List all proposals
  - Query params: `limit`, `offset`, `status`, `agent_name`
  - Returns: List of proposal objects with pagination

- **POST `/api/proposals/{id}/approve`** - Approve a proposal
  - Body: `{ approved_by?, auto_execute? }`
  - Returns: Updated proposal object

- **POST `/api/proposals/{id}/reject`** - Reject a proposal
  - Body: `{ rejected_by?, reason? }`
  - Returns: Updated proposal object

### Metrics

- **GET `/api/metrics/overview`** - Get system metrics
  - Returns: Overview of runs, proposals, agents, recent activity

### Health & Status

- **GET `/health`** - Simple health check
- **GET `/api/status`** - Status with database connectivity check

## Data Models

### Run
Represents a single agent execution/spawn.
```
{
  id: integer,
  agent_name: string,
  agent_type: string,
  status: string (pending|running|completed|failed),
  result: JSON object,
  error: string or null,
  started_at: ISO datetime,
  completed_at: ISO datetime or null
}
```

### Proposal
A proposed action generated by an agent, awaiting approval.
```
{
  id: integer,
  run_id: integer (foreign key),
  agent_name: string,
  proposal_type: string,
  title: string,
  description: string,
  proposed_data: JSON object,
  approval_status: string (pending|approved|rejected),
  approved_by: string or null,
  approved_at: ISO datetime or null,
  created_at: ISO datetime
}
```

### Execution
The execution result of an approved proposal.
```
{
  id: integer,
  run_id: integer (foreign key),
  proposal_id: integer (foreign key),
  status: string (pending|executing|completed|failed|rolled_back),
  result: JSON object or null,
  error: string or null,
  rollback_data: JSON object or null,
  started_at: ISO datetime,
  completed_at: ISO datetime or null
}
```

### AuditLog
Comprehensive audit trail for all system actions.
```
{
  id: integer,
  run_id: integer or null (foreign key),
  action: string,
  agent_name: string,
  actor: string or null,
  details: JSON object,
  created_at: ISO datetime
}
```

## Architecture

### File Structure
```
projects/dashboard/
├── app.py               # Main Flask application & endpoints
├── models.py            # SQLAlchemy ORM models
├── database.py          # Database initialization & utilities
├── requirements.txt     # Python dependencies
└── README.md            # This file
```

### Database Schema
The schema is auto-generated by SQLAlchemy based on the models. Tables are created on first run:
- `runs` - Agent execution records
- `proposals` - Proposed actions
- `executions` - Execution results
- `audit_logs` - Complete audit trail

## Development

### Adding a New Endpoint

1. Add a route to `app.py`:
```python
@app.route('/api/new-endpoint', methods=['GET'])
def new_endpoint():
    # Implementation
    return jsonify({'data': ...}), 200
```

2. Test with curl:
```bash
curl http://localhost:5000/api/new-endpoint
```

### Debugging

Enable Flask debug mode:
```python
# In app.py
if __name__ == '__main__':
    socketio.run(app, host='localhost', port=5000, debug=True)
```

Or set environment variable:
```bash
export FLASK_ENV=development
export FLASK_DEBUG=1
python app.py
```

## Production Considerations

- **Database:** Migrate to PostgreSQL for multi-client deployments
- **Authentication:** Add Flask-JWT-Extended for API authentication
- **CORS:** Configure allowed origins based on your frontend domain
- **Logging:** Use Python logging to file for production
- **Monitoring:** Integrate with CloudWatch or ELK for metrics
- **Secrets:** Use environment variables or AWS Secrets Manager

## Troubleshooting

### Database locked error
- Ensure only one instance of the app is running
- Delete `orchestrator.db` and restart if corrupted

### Port already in use
- Change port in `app.py`: `socketio.run(app, port=5001, ...)`
- Or kill process: `lsof -ti:5000 | xargs kill -9`

### ImportError for Flask modules
- Verify all packages are installed: `pip install -r requirements.txt`
- Check Python version: `python --version` (3.8+)

# Dashboard Specification — Orchestration Platform Monitoring

## Overview

The Dashboard Backend is a Flask-based REST API for monitoring, managing, and auditing the AI orchestration platform. It provides endpoints for tracking agent runs, approving/rejecting proposals, and viewing system metrics.

## Objectives

1. **Visibility** — Real-time insight into agent runs, proposals, and executions
2. **Control** — Approve/reject agent proposals before execution
3. **Auditability** — Complete audit trail of all system actions
4. **Scalability** — Support multiple agents with clean data isolation

## Architecture

### Components

**Backend Stack:**
- **Framework:** Flask (lightweight, minimal overhead)
- **ORM:** SQLAlchemy (database-agnostic, supports SQLite→PostgreSQL migration)
- **WebSocket:** Flask-SocketIO (real-time updates for future frontend)
- **Database:** SQLite for dev, PostgreSQL for production

**Data Models:**
1. **Run** — Agent execution record
2. **Proposal** — Proposed action from agent
3. **Execution** — Result of executing an approved proposal
4. **AuditLog** — Complete event log for compliance

### API Endpoints

#### Runs Management

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/runs` | List runs (paginated) |
| POST | `/api/runs` | Create new run record |
| GET | `/api/runs/{id}` | Get run + related data (proposals, executions, logs) |

**Run Status Flow:**
```
pending → running → completed
             ↓
           failed
```

#### Proposals Management

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/proposals` | List proposals (filterable by status, agent) |
| POST | `/api/proposals/{id}/approve` | Approve proposal, optionally auto-execute |
| POST | `/api/proposals/{id}/reject` | Reject proposal with reason |

**Proposal Status Flow:**
```
pending → approved → [execution]
       ↓
    rejected
```

#### Metrics

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/metrics/overview` | Dashboard overview (counts, recent activity) |

#### Health

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/health` | Simple health check |
| GET | `/api/status` | Status with DB connectivity |

## Data Models

### Run

Represents a single agent spawning and execution lifecycle.

```python
{
  id: int,                          # Primary key
  agent_name: str,                  # e.g., "google-ads-agent"
  agent_type: str,                  # e.g., "google_ads", "crypto"
  status: str,                      # pending | running | completed | failed
  result: JSON,                     # Structured result from agent (optional)
  error: str,                       # Error message if failed (optional)
  started_at: datetime,             # When run began
  completed_at: datetime,           # When run finished (optional)
  
  # Relationships
  proposals: [Proposal],            # Proposals generated by this run
  executions: [Execution],          # Executions from approved proposals
  audit_logs: [AuditLog]           # All events related to this run
}
```

### Proposal

Represents a change/action proposed by an agent.

```python
{
  id: int,                          # Primary key
  run_id: int,                      # Foreign key → Run
  agent_name: str,                  # Which agent proposed this
  proposal_type: str,               # campaign_adjustment | trade | etc.
  title: str,                       # User-friendly title
  description: str,                 # Full explanation
  proposed_data: JSON,              # The detailed proposal payload
  approval_status: str,             # pending | approved | rejected
  approved_by: str,                 # Who approved/rejected (optional)
  approved_at: datetime,            # When decision was made (optional)
  created_at: datetime,             # When proposal was created
  
  # Relationship
  run: Run,
  execution: Execution              # If approved, link to execution
}
```

### Execution

Represents the execution of an approved proposal.

```python
{
  id: int,                          # Primary key
  run_id: int,                      # Foreign key → Run
  proposal_id: int,                 # Foreign key → Proposal
  status: str,                      # pending | executing | completed | failed | rolled_back
  result: JSON,                     # Result of execution (optional)
  error: str,                       # Error message if failed (optional)
  rollback_data: JSON,              # Data to undo execution (optional)
  started_at: datetime,             # When execution began
  completed_at: datetime,           # When execution finished (optional)
  
  # Relationships
  run: Run,
  proposal: Proposal
}
```

### AuditLog

Complete event log for compliance and debugging.

```python
{
  id: int,                          # Primary key
  run_id: int,                      # Foreign key → Run (optional)
  action: str,                      # run_created | proposal_approved | etc.
  agent_name: str,                  # Which agent this action relates to
  actor: str,                       # User who triggered action (optional)
  details: JSON,                    # Contextual data (action-specific)
  created_at: datetime              # When action occurred
}
```

## API Request/Response Examples

### Create a Run

**Request:**
```http
POST /api/runs
Content-Type: application/json

{
  "agent_name": "google-ads-agent",
  "agent_type": "google_ads",
  "status": "completed",
  "result": {
    "campaigns_analyzed": 3,
    "optimization_suggestions": 5
  }
}
```

**Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "agent_name": "google-ads-agent",
    "agent_type": "google_ads",
    "status": "completed",
    "result": { ... },
    "error": null,
    "started_at": "2024-03-04T20:26:00Z",
    "completed_at": "2024-03-04T20:30:00Z"
  },
  "message": "Run 1 created"
}
```

## Integration Points

### With Parent Orchestrator

**Parent spawns agent:**
```python
# Parent: spawn_session(task="...", runtime="subagent")
# Sub-agent completes, returns result
# Parent calls: POST /api/runs
```

**Parent wants to check status:**
```python
# Parent: GET /api/runs?status=running
# Returns all currently running agents
```

**Agent proposes action:**
```python
# Agent: POST /api/runs/123 with status="completed"
# Creates proposals as part of result
# Parent: GET /api/runs/123 to see proposals
# Parent: POST /api/proposals/456/approve (human decision)
```

## Error Handling

### HTTP Status Codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 201 | Created |
| 400 | Bad request (missing fields, invalid data) |
| 404 | Resource not found |
| 500 | Server error |
| 503 | Service unavailable (DB down) |

### Error Response Format

```json
{
  "success": false,
  "error": "Description of what went wrong"
}
```
